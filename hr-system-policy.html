📋 HR系统操作SOP手册

北森(Beisen) · 标准操作流程参考 · 配套全流程排查使用 · 杰德瑞恩J 整理
📌 使用说明
📋
SOP做什么
本手册涵盖9项HR标准操作流程，每项按步骤执行即可。配合全流程排查页使用。
👥
谁用
橙子自动化全体HR（招聘、员工关系、薪酬、绩效）+ IT + 财务相关操作
✅
怎么用
①日常：按SOP步骤操作 ②新人：入职第一周逐项过 ③交接：交接清单+4阶段流程
📖 术语速查
术语定义
北森(Beisen)公司使用的HR SaaS系统，覆盖组织人事、考勤、薪酬、绩效等模块
EHR组织数据北森系统中的组织架构、部门、岗位、员工档案数据（43个部门）
ERP组织对标EHR部门数据与金蝶ERP双组织（200橙子贵州 + 400广东艾富瑞）的对应关系
结构化表单系统内可填写、存储、查询、统计的电子表单（区别于附件上传的Excel/Word）
关键操作指涉及薪酬、组织架构、员工入离职、数据删除/批量修改等影响面大的操作

⚠️ 数据操作红线
🚫 绝对禁止
• 自行编造员工编码、手机号、身份证号
• 批量删除员工档案（无审批）
• 直接修改历史薪酬数据
• 将系统数据以明文方式通过微信/邮件外发
• 使用他人账号操作关键流程
⚠ 需审批后操作
• 批量导入/导出员工数据
• 修改组织架构
• 调整薪资方案/公式
• 删除已归档数据
• 修改系统配置参数
📋 操作SOP 9项核心SOP
SOP-01 新员工入职
① 发起入职流程
在"招聘管理→Offer管理"中确认候选人接受Offer → 点击"发起入职"
责任人：招聘HR
② 信息采集发送
系统自动发送入职信息采集链接 → 候选人填写个人信息（证件、银行、学历、紧急联系人）
责任人：候选人 → 招聘HR催办
③ 信息审核 & 档案创建
HR逐项核对：姓名/身份证号/手机号/银行卡号 → 确认无误后生成员工档案 → 分配工号
责任人：员工关系HR
④ 通知联调
通知IT开通账号（邮件+OA）→ 通知行政准备工位/资产 → 通知直属上级安排工作
责任人：员工关系HR

SOP-02 考勤月结
① 异常数据排查（每月1-3日）
导出考勤异常报表 → 逐条联系员工/上级确认原因 → 系统内完成补卡/修正
责任人：薪酬HR
② 假期余额核对
确认年假、调休、病假等余额与实际情况一致 → 修正系统数据
责任人：薪酬HR
③ 考勤数据冻结（每月4日）
确认考勤数据无误后 → 系统内"月结锁定" → 数据推送至薪酬模块
责任人：薪酬HR
④ 复核确认
HR负责人复核月考勤汇总表 → 签字确认 → 归档
复核人：HR负责人
⚠ 关键提醒：考勤月结截止日为每月4日，逾期未结将延迟全员薪酬核算。若4日为节假日则顺延至下一个工作日。
SOP-03 月度薪酬核算
① 数据准备（每月5-7日）
确认考勤已月结 → 检查当月入离职人员 → 检查异动(调岗/调薪)人员 → 确认社保公积金基数
责任人：薪酬HR
② 系统核算
北森系统内启动"月度薪酬计算" → 自动导入考勤、社保、个税数据 → 生成薪资计算表
责任人：薪酬HR
③ 个别调整 & 审核
如有特殊情况（补发/扣款）手动调整 → 提交HR负责人审核
操作：薪酬HR → 审核：HR负责人
④ 工资条发布（每月10日前）
审核通过后 → 系统生成工资条 → 推送到员工APP/邮箱 → HR确认推送成功
责任人：薪酬HR
🚫 红线：薪酬核算结果在正式发布前属于高度机密。核算期间不得向任何非直接相关人员透露数据。如有员工咨询，统一回复"薪酬发放日可查"。
SOP-04 试用期目标对齐
① 发起（入职后一周内）
绩效HR在系统创建试用期目标表单 → 通知导师和新员工填写
当前状态：以附件上传，待优化为结构化表单
责任人：绩效HR
② 填写目标（当月+次月+下下月）
导师与新员工面谈 → 确定三个月的目标 → 逐月填写 → 双方确认
责任人：新员工 + 导师
③ 月度回顾 & 调整
每月底导师与新员工1v1回顾 → 目标是否需要调整 → 在系统/附件中更新
责任人：导师
④ 试用期结束评估
三个月目标完成情况汇总 → 作为转正评估的核心依据 → 归档
责任人：绩效HR + 导师
🔥 当前待优化：试用期目标对齐应从"附件上传"改为"结构化表单"，实现数据在线填写、存储和追溯。此为P0优先级。
SOP-05 员工离职办理
① 离职申请审批
员工系统提交离职 → 直属上级审批 → HR确认 → 确定最后工作日
发起：员工 → 审批：上级/HR
② 多部门交接流转（并行）
IT：资产回收+账号冻结 → 行政：工位/门禁注销 → 财务：费用清算 → 直属上级：工作交接确认
协调：员工关系HR
③ 薪酬结算（最后工作日次日）
计算当月工作天数薪酬 → 年假折算 → 加班费结算 → 生成离职结算单
责任人：薪酬HR
④ 收尾操作
社保公积金减员（当月15日前操作）→ 出具离职证明 → 员工档案归档 → IT账号注销
责任人：员工关系HR + IT
🚫 时效性红线：社保减员必须在最后工作日当月15日前完成操作！超过15日则次月才能减员，公司需多缴纳一个月社保。
SOP-06 组织架构调整
① 申请
部门负责人提交组织架构调整申请 → 说明调整原因/方案/时间 → HR负责人初审
发起：部门负责人 → 初审：HR负责人
② 审批
HR负责人+管理层审批 → 涉及跨组织（200/400）变更需IT确认ERP同步方案
审批：HR负责人 + 管理层
③ 系统执行（HR+IT并行）
HR：北森系统内修改组织架构/汇报线/岗位编制 → IT：同步更新ERP部门数据
HR：HR负责人 → IT：IT负责人
④ 验证 & 通知
验证EHR与ERP数据一致 → 通知受影响员工和相关部门 → 更新组织架构图
验证：HR + IT → 通知：HR
SOP-07 绩效评估流程
① 方案配置（评估周期开始前）
绩效HR配置评估周期/评分规则/权重/评估人关系 → HR负责人确认
责任人：绩效HR
② 目标录入
员工/上级在系统中录入考核目标 → 双方确认 → 锁定
责任人：员工 + 直属上级
③ 评估打分
自评 → 上级评分 → 隔级确认 → HR汇总 → 校准会（如有）
责任人：全员按角色参与
④ 结果下发 & 面谈
绩效结果下发 → 员工确认 → 面对面绩效沟通 → 结果归档 → （如需）联动薪酬调薪
责任人：直属上级 + HR
SOP-08 月/季度数据报表
① 月度人事报表
每月5日前生成：在岗人数、入离职统计、异动统计 → 发送HR负责人
责任人：员工关系HR
② 月度薪酬报表
每月15日前生成：薪酬汇总、社保公积金汇总、个税申报 → 发送HR负责人+财务Jessie总
责任人：薪酬HR
③ 季度绩效汇总
每季度结束后5个工作日内：绩效分布、试用期转正率、培训完成率 → 发送HR负责人+管理层
责任人：绩效HR
④ 报表归档
所有报表存入腾讯文档指定文件夹，纸质版需HR负责人签字
责任人：各模块HR
SOP-09 招聘管理全流程
① 招聘需求创建与审批（部门→HR→管理层）
部门负责人在北森系统"招聘管理→需求申请"中填写：部门/岗位/编制编号/人数/薪资预算/紧急程度/期望到岗日期 → 提交后走审批链：直属上级→HR负责人→（超预算加财务）→审批通过后关联编制数据
发起：部门负责人 → 审批：直属上级/HR负责人/财务
② 职位发布（审批通过后24小时内）
招聘HR在系统创建职位JD → 选择发布渠道（BOSS直聘/猎聘/内推/校招等）→ 系统自动同步至各渠道并追踪数据 → JD模板由HR负责人统一审核确保合规
责任人：招聘HR
③ 简历筛选 & 面试安排
系统回收简历至人才库 → 招聘HR按JD要求初筛 → 标记状态（通过/淘汰/储备）→ 选定面试官 → 系统发送面试邀请（含会议链接/时间/地点）→ 系统自动同步面试官日历
责任人：招聘HR
④ 面试评价标准
面试官在系统内逐项评分：专业能力/沟通表达/团队匹配/稳定性 → 填写综合评价 → 明确建议（通过/淘汰/待定）→ 多轮面试需汇总全部评价后决策 → 初试淘汰不进入复试
责任人：面试官 → 汇总：招聘HR
⑤ 测评 & 背调（Offer前）
系统发送在线测评（性格/能力/价值观）→ 候选人按时完成 → 系统生成测评报告 → HR审核风险项 → 如有背调需求，HR发起后录入背调结果 → 测评+背调全部通过后方可发起Offer
责任人：招聘HR
⑥ Offer审批与发放
HR在系统创建Offer → 确认薪资/职级/试用期/入职日期/直属上级 → 提交审批（审批链见第五章）→ 审批通过后系统自动发送电子Offer → 候选人线上确认/拒绝 → 确认后自动触发入职信息采集流程
操作：招聘HR → 审批：按审批链
⑦ 招聘数据追踪（每周更新）
HR每周更新招聘漏斗：各岗位投递量/初筛通过率/面试到场率/Offer接受率/人均招聘周期 → 月度汇总报告发送HR负责人+管理层
责任人：招聘HR

🚨 异常处理
🔴 薪酬数据错误
触发：工资条发放后发现金额错误
处理：
1. 薪酬HR立即暂停后续发放
2. 排查原因（系统公式/数据源/人为）
3. 汇报HR负责人+财务Jessie总
4. 制定补发/扣回方案
5. 通知受影响员工（统一口径）
6. 修复系统数据
时限：发现后2小时内响应
🟡 考勤数据缺失/异常
触发：考勤月结时发现大面积数据缺失
处理：
1. 确认是打卡设备问题还是系统同步问题
2. 如是设备问题→联系行政/IT修复
3. 如是系统问题→IT排查日志
4. 启动手工补录流程（需员工+上级确认）
5. 暂缓月结，数据补全后再锁定
🔵 系统宕机/不可用
触发：北森系统长时间不可访问
处理：
1. IT确认问题范围（全员/部分用户）
2. 联系北森技术支持 → 获取恢复时间
3. HR通知各部门：预计恢复时间+替代方案
4. 关键节点（如发薪日）→ 启动手工应急预案
5. 恢复后验证数据完整性
通知模板：腾讯文档存有备查

🔄 HR交接流程 🔥 核心防断层
🚨 问题背景：上一位HR离职后，现HR团队不熟悉系统操作，导致流程断层。此制度专为防止同类问题再次发生。
📋 交接清单（离职HR必交）
1. 系统操作文档 — 按本规范8项SOP逐项录制操作视频或截图流程（至少录前5项）
2. 账号与密码 — 移交所有HR系统及相关第三方平台账号清单（不含密码明文，走IT重置）
3. 待办事项清单 — 列出所有未完成的审批、待处理入离职、进行中的项目
4. 关键联系人 — 北森技术支持、社保局、公积金中心、IT对口人联系方式
5. 数据归档位置 — 腾讯文档/云盘中所有HR相关文件的位置索引
6. 遗留问题清单 — 当前系统/数据中已知但未解决的问题
7. 操作日志 — 最近1个月的关键操作记录摘要
⏱️ 交接流程
阶段① 提前通知（离职前30天）
HR负责人收到离职信息 → 确定接任人选 → 制定交接计划
阶段② 在岗交接（离职前15天）
离职HR按"交接清单"逐项整理 → 带接任HR实操关键流程（至少完成1个完整月结周期）
阶段③ 复核签收（离职前3天）
HR负责人对照清单逐项验收 → 接任HR签名确认 → 离职HR方可办理最后手续
阶段④ 后续跟踪（离职后1个月）
接任HR独立完成一个月度周期 → HR负责人确认无断层 → 交接正式关闭
🔑 关键原则：交接验收不通过 → 离职HR不可办完离职手续。HR负责人有权暂缓《离职证明》出具直至交接完成。
