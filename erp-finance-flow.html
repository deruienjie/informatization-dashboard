2
部分开票 + 金额差 0.1 元
今天刚解决的
📅 2026年5月 · 橙子贵州公司（真实发生）
采购部买了 70个探针模块，5月入库。
⚠️ 供应商只开了其中26个的发票！（剩下44个还没开）
更麻烦的是：发票金额是对的，但金蝶自动生成的应付单比发票多了 0.1 元。
系统自动生成了一条 -0.1 元的差额调整应付单 → 但没有生成成本调整单 → 账不平！

🔴 问题出在哪？

发票（正确）≠ 应付单（多了0.1元）
系统生成了 -0.1 的调整单
但没有自动生成成本调整单

🔵 为什么没生成？

5月入库 + 5月发票 = 同期间
金蝶规则：同期间不自动生成成本调整单
成本直接以发票单价为准核算
跨期间（如5月入库/6月发票）才会自动生成

📊 各账户状态

✅ 存货（原材料）：对的
✅ 发票：对的
✅ 入库凭证：对的
❌ 应付暂估：-0.1元挂在那
❌ 应付对账：不平

🔧 怎么修的？

1. 手工做一笔凭证：
借：主营成本 / 贷：暂估应付（-0.1）
2. 勾选「不应生成」+ 删除凭证
3. 重新审核增值税发票

⚠️ 还能怎么办？他们问了：能不能把应付单金额修改权限放开？
→ 不行。组织间结算的应付单金额 = 应收单金额，强控的。必须先改应收单，应收改完自动反写应付。这是系统安全机制，不是 bug。
3
同期间 vs 跨期间 — 什么时候自动生成成本调整单？
关键差异
同期间 跨期间
举例 5月入库 + 5月发票 5月入库 + 6月发票
成本怎么算 直接以发票单价作为最终成本 5月按入库单价暂估
6月发票到了，差额需调整
自动生成成本调整单？ ❌ 不会 ✅ 会自动生成
入库时 借：原材料 10元
 贷：暂估应付 10元		 借：原材料 10元
 贷：暂估应付 10元
发票时 借：暂估应付 9元（发票价）
 贷：应付账款 9元

成本直接=9元		 借：暂估应付 9元（发票价）
 贷：应付账款 9元

+ 自动生成成本调整单
借：存货 1元（负数）
 贷：暂估应付 1元（负数）
对账结果 暂估对账不平 ← 需手工调整 自动生成调整单 → 对平
💡 一句话记：同期间系统不自动调成本，要手工补凭证；跨期间系统会自动生成成本调整单。判断标准 = 入库月份 ≠ 发票月份 还是 入库月份 = 发票月份。
4
工单结案 — 75颗物料用了66颗，剩下的9颗去哪了？
成本虚高
📅 之前真实案例
生产部领了 75颗物料 去做一个工单。
实际用掉 66颗，9颗没用掉（在生产线上的在制品）。
但工单直接手工结案了！
系统把没用完的9颗全算成了"损耗" → 成本虚高了！
// 应该怎么做：
借：生产成本    66颗 × 单价→ 实际消耗
  贷：原材料   66颗 × 单价→ 从库存减少
  剩余9颗 ← 退库或用在下个工单

// 实际结算时（手工结案的坑）：
系统直接把75颗全部算进生产成本 → 成本虚高了9颗的价格！
⚠️ 教训：工单结案前必须先处理在制品。没用完的物料要么退库、要么转到下个工单。不能直接手工结案——系统会把差额全部当成"损耗"吃掉利润。
5
外币对账 — 欧元为什么永远对不平？
系统设计的坑
📅 真实问题 · 应付科目200欧元
业务系统取的是记账本位币（人民币）
凭证做的是外币（欧元）
→ 对账界面永远两条线，期初永远显示有余额
系统模块用的币种来源 业务系统本位币（人民币）取数来源：入库勾稽表（只能是本位币） 总账凭证欧元财务手工/自动生成的凭证 对账界面按币别分别对人民币和欧元分开显示，无法合并

🔴 为什么平不了？

暂估应付科目的余额来源是入库勾稽表
入库勾稽表只支持记账本位币
所以：暂估应付永远没有外币余额
但对账又是按币别对的 → 无解

🔵 怎么办？

方案1：接受两条线显示（不影响后续账务）
方案2：做一笔总账凭证人工调平
借：应付（本位币）/ 贷：应付（本位币）
但不推荐——调完和直接进本位币没区别

📌 速查：系统凭证自动生成规则

业务环节借贷谁生成的
采购入库原材料/存货应付暂估入库单审核时自动
收到发票应付暂估 + 进项税应付账款发票审核时自动
成本调整(跨期)存货/主营成本暂估应付跨期自动生成调整单
付款应付账款银行存款付款单审核时自动
采购退货应付暂估原材料退货单审核时自动
💡 橙子自动化的特殊设置：应付单不生成凭证。暂估在采购入库环节进，应付单只用来推发票。所以如果应付单金额 ≠ 发票金额，对账就会不平——因为没有凭证兜底。
