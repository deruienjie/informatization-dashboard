05 / 08
为什么必须上 PLM？
因为我们现在的研发管理方式，正在吃掉利润。
📁
图纸到处存
工程师电脑本地存一份、钉盘一份、U盘一份。人走了→图也找不到了。换个工程师接项目，先花一星期找文件。
🔄
BOM 总出错
采购拿到的BOM版本不对，物料买错、多买、少买。上个月就因为BOM版本问题，多采了5万的料堆在仓库。
📱
设变靠钉钉
设计改了一个孔位 → 钉钉群里发一句 → 有人没看到 → 生产按旧图纸做 → 装配时发现对不上 → 返工。
🔌
信息孤岛
研发不知道仓库有多少库存。设计时选了一颗料，采购告诉他"这个供应商早就不做了"，重新设计→耽误两周。
📊
版本混乱
客户设备出问题，售后去查：这个机型交付时的软件版本？硬件图纸版本？BOM版本？——查不到，全靠问人。
🧑‍💻
跨部门靠吼
电气改了一个传感器 → 没通知软件 → 软件按旧型号写程序 → 上机就报错 → 大家都加班排查。
06 / 08
PLM 能做什么？
一句话：把研发"人治"变成"系统治"。
🗄️
文档管理：图纸永远是最新版
所有图纸、BOM、SOP全部存PLM。系统自动锁版本——生产、采购只能拿到"已发布"的最新版本，不会拿错。
📦
BOM管理：从源头不出错
设计师在PLM里搭BOM → 系统自动对比ERP库存 → 提醒"库里有呆滞料可用" → 审批通过后自动推送到ERP，采购再也不用手工录入。
🔄
变更管理：谁改谁知道
设变不再发钉钉。PLM发起ECN → 自动通知电气/软件/采购/生产 → 所有人确认 → 才能生效。不确认？系统锁死，下不了新BOM。
🖥️
CAD集成：工程师不用换工具
SolidWorks里直接操作PLM——画完图一键保存到系统，自动生成BOM。不改变工程师习惯，零学习成本。

⚡ 核心：PLM ↔ ERP 数据打通

设计在PLM完成 → BOM自动推到ERP → 采购按准确BOM下单 → 库存自动扣料 → 成本自动核算。
从设计到成本，一条线跑通，中间没有人工搬运。

🔥 最典型的两个痛点

① 工程师改一颗料，不知道仓库还有50件库存 → 呆滞料损失。PLM上线后：改料时系统自动弹窗"库房有50件存货，确认报废/返工/用完为止？"
② 申请新料号时，不知道库里已有类似物料 → 物料种类膨胀，采购议价能力下降。PLM上线后：系统自动对比"库里已有3个相似轴承，确认申请新号？"
